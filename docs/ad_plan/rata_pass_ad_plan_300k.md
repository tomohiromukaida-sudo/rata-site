## 高意向リード定義

以下のいずれかを満たす登録を「高意向リード」とする。

- `high_intent_score >= 5`（スコア定義は下記）
- または、**1年以内訪問意向あり** かつ **3,000円以上の価格許容あり** かつ **Ocean Gaia体験またはシュノーケル/ダイビングに関心あり**

### スコア定義（high_intent_score）

| 項目 | 加点 |
|---|---|
| visit_intent = すぐに行きたい | +3 |
| visit_intent = 1年以内に行きたい | +2 |
| price_range = 3,000円程度 | +1 |
| price_range = 5,000円程度 | +2 |
| price_range = 10,000円以上でも内容次第 | +3 |
| experience_interest に シュノーケル/ダイビングを含む | +2 |
| interest_area に Ocean Gaiaを実際に見たい を含む | +2 |
| interest_area に 企業/自治体の取り組みとして関心がある を含む | +2 |

`high_intent_score >= 5` で `high_intent = true`。実装は [api/register.js](../../api/register.js) のサーバー側計算を参照。

## ABテスト設計

- 同一地域・同一プラットフォーム内で複数の訴求軸（utm_content）を並行配信し、CTAクリック率・登録完了率を比較する。
- 日本: 5訴求軸（日本初／海の再生／Founding Supporter／徳之島旅行／アーティスト）を同時配信。
- 台湾・香港、シンガポール: 標準型 vs 発見型の2パターンを比較。
- Google検索: キーワード軸を2パターン（「徳之島 アート」系／「Ocean Gaia」指名検索系）で比較。
- 週次レポートでutm_content別の成果を比較し、配信2週目以降に低パフォーマンスの訴求を停止、高パフォーマンスの訴求へ予算を再配分する（実施は人間が行う）。

## 週次運用フロー

1. 週初: 前週の消化金額・登録数・高意向登録数をTursoデータ（または管理画面CSV）から集計し、[weekly_report_template.md](./weekly_report_template.md) に記入する。
2. 訴求軸別・地域別・言語別の成果を比較し、停止候補/強化候補を洗い出す。
3. 人間が広告アカウント上で配信内容（予算配分・クリエイティブのON/OFF）を調整する。
4. 必要に応じて新しい広告コピー案をClaude/Codexに依頼し、人間が入稿する。
5. 翌週分のレポートに反映する。

## 人間が実施する作業

- 支払い設定の確認
- 法務・コンプライアンス確認（特に人物の名前・肖像の使用）
